# BUILD: University Training & Placement Management System - MVP

## Project Overview
Build a full-stack web application for university Training & Placement cells to manage campus placements. This system will have two user roles: T&P Coordinators (admins) and Students.

## Tech Stack Requirements
- **Frontend:** React 18 + Vite + Tailwind CSS
- **Backend:** Node.js + Express
- **Database:** PostgreSQL
- **Authentication:** JWT with httpOnly cookies
- **AI Integration:** Google Gemini 1.5 Flash API for resume optimization
- **File Upload:** Multer for resume PDFs
- **PDF Parsing:** pdf-parse library

## MVP Feature Requirements

### Phase 1: Core Foundation (Build This First)

#### 1. Authentication System
```javascript
// Two user roles: 'coordinator' and 'student'
```

**Coordinator Registration:**
- Email, password, name, university name
- Creates a university record
- Generates unique invite code for students
- Role: 'coordinator'

**Student Registration:**
- Email, password, name
- Roll number (unique per university)
- Branch (dropdown: CSE, IT, ECE, EEE, Mechanical, Civil, Other)

- Role: 'student'

**Login:**
- Email + password
- Returns JWT token
- Redirects based on role

#### 2. Database Schema

```sql
-- Users table
CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  email VARCHAR(255) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  name VARCHAR(255) NOT NULL,
  role VARCHAR(20) NOT NULL CHECK (role IN ('coordinator', 'student')),
  university_id INTEGER REFERENCES universities(id),
  created_at TIMESTAMP DEFAULT NOW()
);

-- Universities table
CREATE TABLE universities (
  id SERIAL PRIMARY KEY,
  name VARCHAR(255) NOT NULL,
  location VARCHAR(255),
  invite_code VARCHAR(50) UNIQUE NOT NULL,
  coordinator_user_id INTEGER REFERENCES users(id),
  created_at TIMESTAMP DEFAULT NOW()
);

-- Students extended profile
CREATE TABLE students (
  id SERIAL PRIMARY KEY,
  user_id INTEGER REFERENCES users(id) UNIQUE NOT NULL,
  roll_number VARCHAR(50) NOT NULL,
  branch VARCHAR(100) NOT NULL,
  graduation_year INTEGER NOT NULL,
  cgpa DECIMAL(3,2),
  active_backlogs INTEGER DEFAULT 0,
  phone VARCHAR(15),
  placement_status VARCHAR(50) DEFAULT 'Not Placed' CHECK (placement_status IN ('Not Placed', 'Placed', 'Opted Out')),
  placed_company VARCHAR(255),
  placed_package DECIMAL(10,2),
  UNIQUE(roll_number, user_id)
);

-- Company drives posted by coordinators
CREATE TABLE company_drives (
  id SERIAL PRIMARY KEY,
  university_id INTEGER REFERENCES universities(id) NOT NULL,
  posted_by_coordinator_id INTEGER REFERENCES users(id) NOT NULL,
  company_name VARCHAR(255) NOT NULL,
  job_role VARCHAR(255) NOT NULL,
  ctc_min DECIMAL(10,2),
  ctc_max DECIMAL(10,2),
  job_description TEXT,
  min_cgpa DECIMAL(3,2) DEFAULT 0.0,
  max_backlogs INTEGER DEFAULT 0,
  allowed_branches TEXT[], -- Array like ['CSE', 'IT', 'ECE']
  registration_deadline TIMESTAMP NOT NULL,
  drive_status VARCHAR(50) DEFAULT 'Active' CHECK (drive_status IN ('Active', 'Completed', 'Cancelled')),
  created_at TIMESTAMP DEFAULT NOW()
);

-- Student registrations for drives
CREATE TABLE student_registrations (
  id SERIAL PRIMARY KEY,
  drive_id INTEGER REFERENCES company_drives(id) NOT NULL,
  student_id INTEGER REFERENCES students(id) NOT NULL,
  registration_status VARCHAR(50) DEFAULT 'Registered' CHECK (registration_status IN (
    'Registered', 'Shortlisted', 'Interview', 'Selected', 'Rejected'
  )),
  resume_id INTEGER REFERENCES resumes(id),
  notes TEXT,
  ai_match_score INTEGER,
  registered_at TIMESTAMP DEFAULT NOW(),
  UNIQUE(drive_id, student_id)
);

-- Student resumes
CREATE TABLE resumes (
  id SERIAL PRIMARY KEY,
  student_id INTEGER REFERENCES students(id) NOT NULL,
  resume_name VARCHAR(255) NOT NULL,
  file_path TEXT NOT NULL,
  resume_text TEXT,
  is_default BOOLEAN DEFAULT false,
  uploaded_at TIMESTAMP DEFAULT NOW()
);

-- AI analysis results
CREATE TABLE ai_analyses (
  id SERIAL PRIMARY KEY,
  student_id INTEGER REFERENCES students(id) NOT NULL,
  drive_id INTEGER REFERENCES company_drives(id) NOT NULL,
  resume_id INTEGER REFERENCES resumes(id) NOT NULL,
  match_score INTEGER,
  missing_keywords TEXT[],
  suggestions TEXT[],
  generated_at TIMESTAMP DEFAULT NOW()
);
```

## Key Implementation Notes

1. **Auth Flow:**
   - Use bcrypt to hash passwords (10 salt rounds)
   - Generate JWT on login with 7-day expiry
   - Store in httpOnly cookie
   - Verify JWT on protected routes

2. **Eligibility Check:**
   - Run on backend when fetching eligible drives
   - SQL query with WHERE conditions for CGPA, backlogs, branch

3. **File Upload:**
   - Use Multer to handle file upload
   - Validate: PDF only, max 5MB
   - Generate unique filename: `{studentId}_{timestamp}.pdf`
   - Store in `/uploads/resumes/`
   - Use pdf-parse to extract text immediately after upload

4. **Gemini Integration:**
   - Set up API key in environment
   - Use gemini-1.5-flash model
   - Handle rate limits (15 req/min)
   - Parse JSON response carefully (handle errors)
   - Cache results in database to avoid re-analysis

5. **Error Handling:**
